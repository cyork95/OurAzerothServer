2. Setting up Borderless Windowed Mode

The original Wrath of the Lich King client does not have a native "Borderless Windowed" checkbox. Running the game in traditional Fullscreen can crash modern Windows PCs or cause issues when switching screens. Maximized Windowed mode acts exactly like borderless fullscreen.

Manual Configuration Settings

  1. Go to your World of Warcraft folder.
  2. Open the **WTF** folder.
  3. Open the file **Config.wtf** in **Notepad**.
  4. Scroll to the bottom and paste the following lines:
SET gxWindow "1"
SET gxMaximize "1"
SET gxRefresh "60"

*Note: If your monitor has a higher refresh rate (e.g. 144Hz or 240Hz), change the "60" value in gxRefresh to match your screen.*

  1. Save the file and launch the game. It will now open as a seamless borderless window.
3. Installing the HD Graphics Pack & 4GB Patch

The server is configured to load beautiful high-definition textures and models if you have installed the optional HD pack.

How to Install the HD Pack

  1. Ensure your downloaded HD pack files end with .mpq (e.g. patch-W.mpq).
  2. Move these patch files into your WoW client's **Data/** folder (NOT the enUS subfolder, directly in Data/).
  3. Important: Do not overwrite or rename files named patch.mpq, patch-2.mpq, or patch-3.mpq. Rename your custom HD patches to any unused letter or number (e.g. patch-W.mpq, patch-K.mpq, or patch-5.mpq).
  4. Delete the **Cache/** directory inside your WoW folder before launching. This folder is safely rebuilt by the game and prevents graphics glitches.
⚠️ CRITICAL: HD Graphics Crashes & The 4GB Patch

WotLK was released in 2008 as a 32-bit application. Windows strictly limits 32-bit applications to **2 GB of RAM**.

Because the HD Pack loads extremely large textures, changing your display options or walking into populated zones will quickly exceed 2 GB, causing your game to crash instantly (usually throwing **Error #132**).

The Fix: Download the free utility tool here: Download NTCore 4GB Patch Tool. Open the utility, select your Wow.exe file, and apply the patch. This allows the game to use up to 4 GB of RAM, fully preventing any HD graphics-related memory crashes.

🏹 5. Hunter Melee Weaving Disconnect Workaround

If you are playing a Hunter (like Eliphas), you may experience sudden disconnects to the login screen right when clicking Raptor Strike in melee range while Auto Shot is active.

This is a known client-side stability bug in the original 2008 WotLK client's auto-attack stance toggling logic.

How to Fix in Your Client Settings

  1. Press Esc to open the Game Menu.
  2. Select Interface.
  3. Select the Combat category on the left side.
  4. Find the setting Auto Attack/Auto Shot:
    • Uncheck the box if it is currently enabled.
  5. Alternatively, ensure Stop Auto Attack is checked.
  6. Click Okay and test using Raptor Strike again in melee range.

*Note: Disabling this prevents the client from automatically toggling combat modes on its own (you will simply right-click the mob or click a melee ability to start melee swings, preventing client-side combat state locks).*

Creating Custom Follower Companions

Instead of relying purely on the random bots running around the world, you can design your own custom companion characters who will follow you, dungeon with you, and help you quest.

1. Use the Custom Follower Creator

Go to the Admin Control Panel and scroll to the bottom to find the **AI Custom Follower Creator** card. Fill in your character's name, choose a name for your follower, choose their race/class/gender/level, and click **Create Follower**.

⚠️ Server Restarts & In-Memory Cache

The game server loads all character files into an in-memory cache when starting up. When you create a follower through the web dashboard, they are saved directly in the database, but the running game server won't know they exist yet.

Before you can summon them, you must restart the server. You can do this by clicking "Reload Conf" or "Restart Services" in the Admin Panel, or by typing .server restart 5 in your game chat. The server takes about 10 seconds to reboot.

3. Summoning & Gearing Your Follower

After creating your follower and restarting the server, log in on your main character and perform the following steps:

How to Bring Them Into the World (Summoning)

  1. Type either of the following commands in your main chat bar to load the bot and add them to your party:
.playerbots bot add FollowerName

*(Alias: .pb bot add FollowerName. Replace FollowerName with your bot's name).*

  1. The bot will spawn directly next to you and join your party.

How to Equip Them (Gearing)

With our server's Altbot Command Bypass, you can instantly gear your companion directly from the chat bar without having to whisper them:

.playerbots bot init=auto FollowerName

*(Alias: .pb bot init=auto FollowerName. Replace FollowerName with your bot's name. You can also specify other gear qualities like rare, epic, or legendary).*

This triggers the server's bot factory to instantly generate and equip them with a full set of appropriate level-scaling gear! As you level up, you can run this command again to refresh their gear to match your level.

Playerbot In-Game Commands Reference

To control your bot companion, you must communicate with them. You can give commands in two ways:

  • Group Chat: Type the command in party chat (e.g. /p follow or /p stay) and all bots in your group will obey.
  • Whispers: Send a private whisper to a specific bot (e.g. /w BotName follow) to command only that bot.
💡 Pro Tip: Quick Invites

Instead of manually typing invite commands, you can click the "Invite" button next to any online bot in the Admin Panel. If the browser blocks clipboard access, copy the command displayed in the popup prompt and paste it in-game.

Talking to Bots (Ollama AI Chat Integration)

The server is integrated with a local Large Language Model (Ollama). This allows the playerbots in the game world to behave like real players, complete with unique personalities, factions, and area awareness. You can have conversations with them!

How to Chat with Bots

  • Say Chat (`/s [Message]`): Stand near a bot in the world and type in local Say chat. For example, if you stand near Dianne, type /s Hey Dianne, nice weather today. There is a **90% chance** she will reply back to you privately or in local chat!
  • Party Chat (`/p [Message]`): If you invite bots to a group, they will listen to party chat. If you say something to the group, bots have a **90% chance** to reply, joke, or chime in on your conversation.
🔒 Why Whispers Do Not Trigger AI Chat

Whisper replies are disabled for Ollama. Whispering a bot (e.g. /w BotName hello) will not generate an AI response.

Reason: Whispers are strictly reserved for playerbot control commands (like follow, stay, init=auto). If Ollama intercepted whispers, the AI would try to carry on a conversation instead of letting you control their movement and gear!

To chat with a bot, always use local Say or group Party chat.

Enabled Modules & Server Customizations

Our server has several custom modules installed to enhance single-player progression. Here is an overview of what they do:

1. Autobalance (Dungeon & Raid Scaling)

Normally, dungeons in World of Warcraft require a full group of 5 players, and raids require 10 to 40 players. The Autobalance module dynamically scales the health, damage, and armor of bosses and monsters based on your current party size. If you want to run Deadmines solo or with just 1 or 2 bot companions, the monsters will scale down so you can complete the dungeon comfortably.

2. AHBot (Auction House Economy Simulator)

To make the auction house feel alive on a single-player server, the AHBot module simulates a live economy. It automatically posts trade goods, equipment, recipes, and materials to the Auction House, and will buy items that you post for a fair price. This allows you to sell loot and buy resources just like you would on a populated official server.

3. Level Synchronization

Instead of running into level 80 bots while your own character is level 10, the server has Level Synchronization enabled. The maximum level of random bots in the world is dynamically capped by the highest level online real human player. As you progress and level up, the bots in the world will level up alongside you.

4. Transmog (Transmogrification features)

The Transmog module allows you to change the appearance of your armor and weapons to match other items you've collected, without losing the stats of your current equipment. Visit a Transmogrifier NPC in any major city to customize your character's look!

5. Diablo-Style Random Enchants

The Random Enchants module simulates a Diablo-style loot system. When equipment drops from monsters, is looted from chests, or is crafted, there is a random chance it will automatically roll bonus magical stats and enchants (ranging from simple stat boosts to class-specific spell modifiers). This adds an exciting element of RNG to equipment hunting!

6. Challenge Modes (Opt-in Gameplay Rules)

The Challenge Modes module introduces custom gameplay rulesets to the server. These challenges are NOT realm-wide; instead, they are character-specific opt-ins. When you create a new Level 1 character, you will find a mysterious interactive object called the Shrine of Challenge (a glowing blue chalice object) in your starting area. If you interact with it at Level 1 before gaining any XP, you can choose to activate one or more of these rulesets for that character. The available modes are:

  • Hardcore Mode: If your character dies, you permanently become a ghost and can never be revived (one-life run).
  • Semi-Hardcore Mode: When you die, you lose all worn equipment and carried gold, but can still revive.
  • Ironman Mode: Enforces strict Ironman rules (wear only white/gray gear, no talents, no professions, no buffs, no groups, and permanent death on first defeat).
  • Self-Crafted Mode: You are only allowed to wear equipment that your character has personally crafted.
  • Item Quality Mode: You are only allowed to wear equipment of Normal (White) or Poor (Gray) quality.
  • Slow XP Modes: Reduces your experience gains to 0.5x (Slow) or 0.25x (Very Slow) for a slower level progression.
  • Quest XP Only: You gain experience points exclusively from completing quests; killing monsters yields zero XP.

7. AoE Looting System

The AoE Loot module allows you to loot all nearby fallen monsters at the same time by clicking on just one of them, saving you from clicking on corpses individually.

8. Mythic+ Dungeons

The Mythic+ Dungeons module introduces key-based dungeon difficulty scaling. Interact with the pedestal at the start of any dungeon using a Mythic Keystone to enable scaling health, damage, and active gameplay affixes (like Bolstering or Sanguine) for a high-end challenge.

9. Solo LFG System

The Solo LFG module enables solo players to queue for the Dungeon Finder system. Once you queue, the system will match you or automatically populate the party so you can run dungeons without waiting for other real players.

10. All-Professions & Free Trainer

The Free Professions module spawns NPC trainers who teach professions for free. Additionally, the server's trade skill limit has been modified to allow every character to learn all 11 primary professions simultaneously (e.g. you can be a Blacksmith, Engineer, Alchemist, Enchanter, and Gatherer all on the same character!).

11. All Races All Classes (ARAC)

The ARAC module removes the traditional racial restrictions on class selection. This allows you to create any race and class combination you desire, such as a Tauren Rogue, a Gnome Paladin, or an Undead Druid. These combinations are fully supported on both the server and client (via required patch) and include properly scaled stats and starting equipment.

12. Account-Wide Mounts & Achievements

The Account Mounts and Achievements modules synchronize all learned mounts and unlocked achievements across all characters on your account. If you learn a rare mount on one character, it becomes immediately usable by all your alt characters.

13. Item Upgrade System

The Item Upgrade module spawns an NPC in major cities who can upgrade the item level and stats of your equipment in exchange for gold or custom currencies.

🔧 Reverse Engineering Guidelime for WotLK 3.3.5a

To bridge the modern Guidelime addon APIs down to the legacy WotLK client, we injected custom compatibility patches and shims directly into the client files:

💾 UIPanelScrollFrame Auto-Naming

Anonymous scroll frames fail on WotLK if they lack a global name. We hooked the global CreateFrame inside LibStub to automatically assign unique names to anonymous scroll frames and bind their scrollbar properties.

🚀 Memory Leak Optimization

The original CreateFrame hook allocated unique closures for .SetEnabled for every frame, triggering fatal client memory allocation crashes ("memory block too big"). We optimized it to bind a single static shim reference.

⚙️ C_QuestLog.GetQuestInfo Fallback

WotLK lacks the modern quest log info APIs. We shimmed C_QuestLog.GetQuestInfo to safely return nil, forcing Guidelime to fall back to Questie's WotLK query wrapper (QUESTIE.getQuestName).

🗺️ Vector Coordinates & Mapping

To allow HereBeDragons-2.0's vector math to run on WotLK's flat 2D map coordinates, we shimmed CreateVector2D and injected coordinate GetXY() methods into compatibility files.
